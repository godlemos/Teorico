# Término Definición
1HTMLLenguaje de marcado estándar que estructura y presenta contenido en la web, usando etiquetas para definir elementos como párrafos, encabezados y listas.
2CSSLenguaje de hojas de estilo que controla la presentación visual de documentos HTML, permitiendo personalizar colores, tipografías, espaciados y animaciones.
3JavaScriptLenguaje de programación que añade interactividad y funcionalidad dinámica a las páginas web, permitiendo crear eventos, manipular el DOM y realizar operaciones asíncronas.
4PHPLenguaje de programación del lado del servidor que genera contenido dinámico en tiempo real, procesa formularios y maneja datos de bases de datos.
5MySQLSistema de gestión de bases de datos relacional que almacena y organiza datos en tablas relacionadas, permitiendo consultas complejas y transacciones seguras.
6XMLLenguaje de marcado extensible que define reglas para codificar documentos en un formato que es tanto legible para humanos como para máquinas.
7DOMModelo de objetos que representa la estructura de un documento HTML o XML, permitiendo manipular elementos y contenido dinámicamente.
8JSONFormato ligero y portable para intercambio de datos estructurados, basado en texto y fácilmente legible tanto para humanos como para máquinas.
9APIConjunto de reglas y protocolos que permiten que diferentes aplicaciones o servicios se comuniquen y compartan datos entre sí.
10AJAXTecnología que permite actualizar partes específicas de una página web sin necesidad de recargar toda la página, mejorando la experiencia del usuario.
11GETMétodo HTTP que solicita datos al servidor, ideal para búsquedas y consultas que no modifican el estado del servidor.
12POSTMétodo HTTP que envía datos al servidor para ser procesados, ideal para transacciones que requieren seguridad como envío de formularios.
13isset()Función PHP que verifica si una variable existe y no es null, útil para manejar datos de formularios y sesiones.
14echoFunción PHP que imprime texto o variables directamente en la salida del navegador, comúnmente usada para mostrar contenido dinámico.
15letPalabra clave en JavaScript que declara variables con alcance de bloque, permitiendo mejor control sobre el ámbito de las variables.
16constPalabra clave en JavaScript que declara constantes, valores que no pueden ser modificados después de su declaración.
17varDeclaración de variable en JavaScript que tiene alcance global o de función, aunque está siendo reemplazada por let y const.
18SESSIONMecanismo que mantiene datos del usuario durante su interacción con un sitio web, permitiendo una experiencia personalizada y segura.
19COOKIEPequeño archivo de texto que almacena información del usuario en el navegador, permitiendo recordar preferencias y estados entre visitas.
20functionBloque de código reutilizable que realiza una tarea específica, que puede ser llamado múltiples veces con diferentes parámetros.
21arrayLista ordenada de elementos que pueden ser de diferentes tipos, permitiendo almacenar y acceder a múltiples valores de manera eficiente.
22whileBucle que se ejecuta mientras una condición sea verdadera, ideal para operaciones que requieren ejecución indefinida hasta que una condición se cumpla.
23forBucle que itera sobre una secuencia de números o elementos, ideal para operaciones repetitivas con un número fijo de iteraciones.
24if/elseEstructura de control que ejecuta código basado en condiciones booleanas, permitiendo tomar decisiones en el flujo del programa.
25selectElemento HTML que muestra una lista desplegable de opciones, permitiendo al usuario seleccionar un valor de una lista predefinida.
26inputElemento HTML que permite al usuario ingresar datos de diferentes tipos como texto, números, fechas o archivos.
27buttonElemento interactivo que puede disparar eventos o enviar formularios, comúnmente usado para acciones como enviar, guardar o cancelar.
28formElemento HTML que recopila datos del usuario a través de campos de entrada, selectores y botones, permitiendo el envío de información al servidor.
29spanContenedor en línea que no afecta el flujo del documento, ideal para aplicar estilos o manipular texto específico dentro de un párrafo.
30divContenedor en bloque que agrupa elementos relacionados, permitiendo aplicar estilos y estructurar el contenido de manera lógica.
31classAtributo HTML que asigna una clase a un elemento, permitiendo aplicar estilos CSS y selectores de JavaScript a múltiples elementos.
32idAtributo HTML que identifica de manera única un elemento en el documento, permitiendo seleccionarlo específicamente para estilos o manipulación.
33hrefAtributo que especifica la URL de destino de un enlace, permitiendo navegar entre diferentes páginas o recursos web.
34srcAtributo que indica la ubicación de un recurso externo como una imagen, video o script que debe ser cargado en la página.
35altAtributo que proporciona texto alternativo para imágenes, mejorando la accesibilidad y SEO al describir el contenido visual.
36metaElemento que define metadatos sobre el documento como el conjunto de caracteres, descripción y palabras clave.
37SEOConjunto de técnicas y estrategias para mejorar la visibilidad de un sitio web en los resultados de búsqueda de motores como Google.
38IPNúmero único que identifica una computadora o dispositivo en una red, permitiendo el enrutamiento de datos en Internet.
39DominioNombre legible que hace referencia a una dirección IP, permitiendo a los usuarios acceder a sitios web usando nombres memorables.
40DNSSistema que traduce nombres de dominio legibles por humanos en direcciones IP numéricas para que las computadoras puedan comunicarse.
41ServidorComputadora que almacena y sirve recursos web, procesa solicitudes de clientes y entrega contenido solicitado.
42HostProveedor de servicios que almacena y mantiene sitios web en servidores, permitiendo que estén accesibles en Internet.
43HTTPSProtocolo seguro que combina HTTP con SSL/TLS para proporcionar una conexión cifrada y autenticada entre cliente y servidor.
44SSLProtocolo de seguridad que cifra la comunicación entre cliente y servidor, protegiendo datos sensibles durante el tráfico web.
45BackendParte invisible del sitio web que maneja la lógica de negocio, procesamiento de datos y funcionalidad del servidor.
46FrontendParte visible y accesible del sitio web que interactúa directamente con los usuarios, incluyendo la interfaz de usuario y la experiencia visual.
47FrameworkConjunto de herramientas y bibliotecas que proporcionan una estructura predefinida para desarrollar aplicaciones web de manera más eficiente.
48CMSSistema que permite gestionar y publicar contenido web sin necesidad de conocimientos técnicos avanzados, facilitando la creación y actualización de sitios web.
49URLIdentificador único que especifica la ubicación exacta de un recurso en Internet, incluyendo el protocolo, dominio y ruta.
